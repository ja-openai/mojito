# Mojito ICT Extension

Vite + TypeScript rewrite of the Mojito in-context translation Chrome extension.

## Development

```sh
cd chromeextension
source ../webapp/use_local_npm.sh
npm install
npm run typecheck
npm run build
```

Load the unpacked extension from `chromeextension/`. The manifest points to generated files under
`dist/`.

For local fixture testing, run:

```sh
npm run fixture
```

Then open `http://localhost:8123/ict.html` or `http://127.0.0.1:8123/ict.html`. The fixture server
only serves the instrumented page. The popup default Mojito base URL points at the dev server
through Vite, so MT calls go to:

```text
http://localhost:5173/api/machine-translation
```

Enable ICT and MT preview in the popup to verify that the `MT_REQUIRED` fixture string is replaced
with server MT output when the backend has a machine translation engine configured.

## Notes

- Source lives under `src/`.
- `dist/` is generated by Vite and is not committed.
- The rewrite keeps the current ICT wire format: Unicode control delimiters around tags-block
  metadata in rendered strings.
